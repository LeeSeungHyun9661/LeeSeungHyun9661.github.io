자바 문법 구성

자바 언어의 기초적인 문법에 대해 알아보도록 합시다. 자바 문법을 구성하는 요소는 클래스, 인스턴스, 변수, 메서드 그리고 주석입니다.

  1. 클래스(Class):객체지향 프로그램의 기본 구조로 자바에서 모든 프로그램 소스는 클래스 단위로 시작하게 됩니다.
    • 프로그램 소스는 .java 파일이고 컴파일된 결과는.class가 됩니다.
    • 일반적으로 클래스 이름과 소스파일명은 동일합니다.
    • 대부분의 경우 프로그램은 여러 클래스로 구성되며 실행을 위해서는 main() 메서드가 필요합니다.
  2. 인스턴스(Instance):클래스로 부터 생성된 객체로 클래스는 객체를 정의한 틀이고 실제 프로그램은 인스턴스를 통해 동작하게 됩니다.
    • main() 메서드는 단지 프로그램을 실행하는 진입점이고 실제 클래스를 사용하려면 new() 연산을 통해 인스턴스를 생성해야 합니다.
    • main()에서 클래스부에 선언된 변수(멤버)를 접근할 수 없으며 인스턴스를 통해 사용해야합니다.(인스턴스 변수)
    • 인스턴스에서 변수와 메서드 사용은 인스턴스명.변수명, 인스턴스명, 메서드명과 같은 형식으로 사용
  3. 변수(Variable):일반적인 프로그램언어의 변수와 같은 개념으로 하나의 값을 저장할 수 있는 저장공간을 뜻합니다.
  4. 메서드(Method):일반적인 프로그램언어의 함수와 유사합니다. 함수는 단순한 기능을 모듈화 한것이지만 메서드는 객체의 동작(행위)을 정의 합니다.
  5. 주석(Comment):프로그램 실행에 아무 영향도 주지 않습니다. 코드에 대한 여러 설명과 메모의 역할을 합니다. 올바른 주석 처리는 코드의 가독성을 높여줍니다.
// main.java 파일의 main 클래스 => 파일 이름과 클래스 이름은 동일해야 한다!
  public class main { 
  // main 클래스에 있는 실행을 위한 메서드 => 코드 실행을 위해서는 반드시 필요하다.
  public static void main(string[] args) { 
      int number = 1; // 정수형 변수          
      string hello = "hello world!"; // 문자열 변수
      
      number = addfive(number); // 메서드 호출과 반환값 저장
  }

  static int addfive(int number) { // 메서드
      return number + 5; // 반환값
  }
}

식별자 규칙

각 구성 요소에 대해 식별자는 규칙에 따라 작성되어야 합니다. 이미 지정되어 있는 식별자 코드와 충돌하지 않도록 하기 위해서입니다. 어떠한 코드들은 이미 그 이름 자체에 기능이 부여되어 있습니다.

규칙 설명 예시
첫 문자 문자, _ 또는 $로 시작되어야 하며 숫자로 시작할 수 없다. validIdentifier, _underscore, $dollar
그 외 문자 문자, 숫자, _ 또는 $로 구성될 수 있다. identifier1, another_Identifier, $identifier2
예약어 자바의 예약어는 식별자로 사용할 수 없다. int, class, if (사용 불가)
대소문자 구분 자바의 식별자는 대소문자를 구분한다. variable, Variable (다른 식별자)
길이 제한 없음 식별자의 길이는 제한이 없고 공백은 포함할 수 없다. thisIsAQuiteLongIdentifier, no SpacesAllowed

이러한 규칙은 반드시 지켜져야 합니다. 반면에 지켜지도록 권장하는 관례 또한 존재합니다.

관례 설명 예시
클래스 이름 대문자의 명사로 시작 Customer, Order, Product
메서드 이름 소문자의 동사 calculateTotal(), getCustomerName(), sendEmail()
변수 이름 소문자의 명사 total, customerName, email
상수 이름 대문자의 명사 MAX_VALUE, MIN_WIDTH, DEFAULT_TIMEOUT

자바의 간단한 코드를 작성해봅시다. 자바는 실행을 위해서 main 메서드가 필요합니다. 다음과 같은 코드가 있습니다.

변수의 종류와 선언

변수는 프로그래밍 언어의 벽돌과 같습니다. 벽돌 없이 집을 지을 수 없듯 변수 없이 의미있는 프로그램을 만들기는 어렵습니다. 변수란 값을 저장하기 위한 공간이며 다양한 타입을 가지고 있습니다. 크게 두 가지로 나눌 수 있습니다.

기본형 (Primitive Types)

기본형 변수는 자바의 가장 기본적인 데이터 타입으로, 값 자체를 저장합니다. 총 8가지가 있습니다.

데이터 타입 크기 값의 범위 리터럴 표현
byte 8비트 -128에서 127까지의 값을 저장합니다. 예: 123
short 16비트 -32,768에서 32,767까지의 값을 저장합니다. 예: 12345
int 32비트 -2,147,483,648에서 2,147,483,647까지의 값을 저장합니다. 예: 1234567890
long 64비트 -9,223,372,036,854,775,808에서 9,223,372,036,854,775,807까지의 값을 저장합니다. 예: 123456789012L
float 32비트 부동 소수점 숫자를 저장합니다.
±1.4E-45에서 ±3.4028235E38까지의 범위를 가집니다.		 예: 3.14F
double 64비트 배정도 부동 소수점 숫자를 저장합니다.
±4.9E-324에서 ±1.7976931348623157E308까지의 범위를 가집니다.		 예: 3.14159265359
char 16비트 단일 문자를 저장합니다. 유니코드 문자를 지원하며, 작은따옴표(' ')로 표현합니다. 예: 'A'
boolean 1비트 true 또는 false 값을 저장합니다. 예: true 
public class primitivetypesexample {
  public static void main(string[] args) {
      byte a = 10;
      short b = 200;
      int c = 1000;
      long d = 10000l;
      float e = 3.14f;
      double f = 3.14159;
      char g = 'a';
      boolean h = true;
  }
}

참조형 (Reference Types)

참조형 변수는 객체의 메모리 주소를 저장합니다. 참조형 변수는 클래스, 배열, 인터페이스 등과 같이 사용자 정의 타입을 포함합니다.

타입 설명 및 코드 예시
클래스

객체 지향 프로그래밍의 핵심으로, 클래스는 데이터와 메서드를 포함할 수 있습니다. 참조형 변수는 클래스의 인스턴스를 가리킵니다.

public class person {
  string name;
  int age;    
  public person(string name, int age) {
      this.name = name;
      this.age = age;
  }
}
public class referencetypeexample {
  public static void main(string[] args) {
      person person = new person("john", 30);
  }
}
배열

동일한 타입의 변수들을 모아놓은 자료구조입니다. 배열은 고정된 크기를 가지며, 인덱스를 통해 접근합니다.

public class arrayexample {
  public static void main(string[] args) {
      int[] numbers = {1, 2, 3, 4, 5};
  }
}
인터페이스

인터페이스는 클래스가 구현해야 하는 메서드의 집합을 정의합니다. 인터페이스 타입 변수는 이를 구현한 클래스의 객체를 참조할 수 있습니다.

interface animal {
  void makesound();
}

class dog implements animal {
  public void makesound() {
      system.out.println("woof");
  }
}

public class interfaceexample {
  public static void main(string[] args) {
      animal dog = new dog();
      dog.makesound();
  }
}

기본형과 참조형을 구분하는 이유

기본적으로 프로그램은 메모리에 변수를 쓰고 읽으며 값을 저장하고 불러오게 됩니다. 자바는 이러한 변수의 저장 공간을 스택(Stack)과 힙(Heap)으로 나눕니다.

구분 설명
스택(Stack)

높은 주소에서 낮은 주소의 방향으로 할당됩니다. 즉 아래에서 위로 쌓아올리는 형식으로 가장 늦게 저장된 데이터가 가장 먼저 인출됩니다. 따라서 크기가 제한적이지만 빠르게 접근할 수 있습니다.
힙(Heap)

스택과 반대 방향으로 주소가 할당됩니다. 상대적으로 접근이 느리고 메모리를 관리(변수를 할당하고 해제하는 과정이 필요합니다.)가 필요하지만 동적으로 할당할 수 있습니다.

이러한 두 공간의 특징을 활용하기 위해 지역 변수와 메서드 호출에 따른 생성 변수에 빠르게 접근하고 자동으로 메모리를 할당할 수 있는 스택에는 기본형 변수를 저장합니다. 반면에 동적으로 할당할 수 있어 배열과 객체를 저장할 수 있는 힙에는 참조형 변수값을 저장하게 됩니다.

따라서 우리가 선언하는 인스턴스나 배열은 값이 아닌 힙 공간에 위치한 값의 주소를 의미합니다. 이에 따라 메서드에 값을 전달하거나 두 값을 복사하는데 있어 참조형 변수와 기본형 변수가 다른 결과를 보여주기도 합니다. 이후 깊은 복사와 얕은 복사를 통해 더 깊게 알아보도록 합시다.

형 변환

변수의 타입은은 선언과 함께 결정됩니다. 하지만 형 변환을 사용해 타입을 변경하는 방법도 있습니다. 문자열을 정수로, 정수를 실수로 변환할 수 있습니다.

  1. 묵시적 형 변환(Implicit Casting)

    묵시적 형 변환은 변환에 대한 특별한 지시 없이 그 형을 변환시키는 것을 의미합니다. 변환하는 타입의 크기가 더 클 경우 데이터의 손실이 없기 때문에 안전하게 그 데이터를 변환할 수 있습니다.

    int intval = 100;
    long longval = intval;  // int에서 long으로 묵시적 형변환
    float floatval = longval;  // long에서 float로 묵시적 형변환

  2. 명시적 형변환 (Explicit Casting)

    반면에 크기가 더 작은 형태로 변환하는 경우 데이터가 변경될 가능성이 있습니다. 따라서 명시적으로 데이터 형변환을 해줄 필요가 있습니다.

    double doubleval = 100.04;
    int intval = (int) doubleval;  // double에서 int로 명시적 형변환 (소수점 이하가 잘림)

    다음과 같이 (타입)으로 그 형태 변화를 할 수 있습니다.

여기까지의 내용은 수치형 데이터나 수치를 기반으로 하는 문자(ASCII) 형태의 데이터에 해당합니다. 하지만 문자열은 다릅니다. 문자열은 기본형이 아닌 참조형 변수이고 형변환을 위해서는 다른 방법을 사용해야 합니다.

문자열 -> 정수

문자열의 경우 Integer.parseInt(문자열)을 활용해 변환 

string s = "12345";
        int i = integer.parseint(s);

정수 -> 문자열

정수의 경우 Integer.toString(정수)을 활용해 변환 

int i = 12345;
    string s = integer.tostring(i);

변수의 생명주기

vmfh

앞서 컴파일에 대해 설명하며 변수가 메모리에 저장된다는 것을 알려드렸습니다. 프로그램이 실행되면 메모리에 변수가 저장되고 프로그램이 종료되면 변수의 공간도 반환됩니다. 이 과정중에 변수는 하나의 생명처럼 태어나서 영향을 행사하고 이후 소멸됩니다. 어디서 태어나 어디까지 영향을 끼치는지에 따라 종류가 나누어집니다.

종류 설명
인스턴스 변수
(Instance Variables)
  • 정의: 클래스의 인스턴스에 속하는 변수
  • 선언 위치: 클래스 블록 내, 메서드 외부
  • 생명 주기: 객체 생성 시 ~ 가비지 컬렉션 시
  • 접근 방법: 객체를 통해 접근
public class Person {
    // 인스턴스 변수
    String name;
    int age;
}
클래스 변수
(Class Variables)
  • 정의: 클래스에 속하며 모든 인스턴스가 공유, static 키워드 사용
  • 선언 위치: 클래스 블록 내, 메서드 외부, static 키워드 포함
  • 생명 주기: 프로그램 시작 시 ~ 종료 시
  • 접근 방법: 클래스 이름 또는 객체를 통해 접근
public class Person {
    // 클래스 변수
    static int population;
}
지역 변수
(Local Variables)
  • 정의: 메서드, 생성자 또는 블록 내에서 선언된 변수
  • 선언 위치: 메서드, 생성자 또는 블록 내부
  • 생명 주기: 블록 실행 시 ~ 블록 종료 시
  • 접근 방법: 선언된 블록 내부에서만 접근
public void display() {
    // 지역 변수
    int count = 0;
    System.out.println(count);
}
매개 변수
(Parameters)
  • 정의: 메서드나 생성자의 매개 변수 목록에 선언된 변수
  • 선언 위치: 메서드 또는 생성자의 선언부
  • 생명 주기: 메서드 호출 시 ~ 종료 시
  • 접근 방법: 해당 메서드 또는 생성자 내부에서 접근
public void setName(String name) {
    // 매개 변수
    this.name = name;
}

연산자

연산자는 프로그램에서 데이터를 처리하고 결과를 산출하기 위한 연산의 과정에 사용하는 기호입니다.

  1. 산술 연산자
    연산자 설명 예시
    + 덧셈 int sum = 5 + 3;
    - 뺄셈 int difference = 5 - 3;
    * 곱셈 int product = 5 * 3;
    / 나눗셈 int quotient = 6 / 3;
    % 나머지 연산 int remainder = 6 % 4;
  2. 비교 연산자
    연산자 설명 예시
    == 동등 비교 boolean isEqual = (x == y);
    != 부등 비교 boolean isNotEqual = (x != y);
    > 초과 비교 boolean isGreater = (x > y);
    < 미만 비교 boolean isLess = (x < y);
    >= 이상 비교 boolean isGreaterOrEqual = (x >= y);
    <= 이하 비교 boolean isLessOrEqual = (x <= y);
  3. 논리 연산자
    연산자 설명 예시
    && 논리 AND boolean result = (condition1 && condition2);
    || 논리 OR boolean result = (condition1 || condition2);
    ! 논리 NOT boolean result = !condition1;
  4. 대입 연산자
    연산자 설명 예시
    = 대입 int x = 5;
    += 덧셈 후 할당 x += 3;
    -= 뺄셈 후 할당 x -= 3;
    *= 곱셈 후 할당 x *= 3;
    /= 나눗셈 후 할당 x /= 3;
    %= 나머지 연산 후 할당 x %= 3;
  5. 비트 연산자
    연산자 설명 예시
    & 비트 AND int result = x & y;
    | 비트 OR int result = x | y;
    ^ 비트 XOR int result = x ^ y;
    ~ 비트 NOT int result = ~x;
    << 왼쪽으로 이동 int result = x << 1;
    >> 오른쪽으로 이동 int result = x >> 1;
  6. 단항 연산자
    연산자 설명 예시
    + 단항 덧셈 int result = +x;
    - 단항 뺄셈 int result = -x;
    ++ 증가 연산자 x++;
    -- 감소 연산자 x--;
    ! 논리 NOT boolean result = !condition;
    ~ 비트 NOT int result = ~x;
