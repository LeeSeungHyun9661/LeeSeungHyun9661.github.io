Java Grammar Composition

Let's explore the basic grammar of the Java language. The components that make up Java syntax include classes, instances, variables, methods, and comments.

  1. Class: The fundamental structure of object-oriented programs, where every Java program starts at the class level.
    • Java program sources are .java files, and compiled results become .class files.
    • Generally, the class name and source file name are the same.
    • In most cases, programs are composed of multiple classes, and an main() method is required for execution.
  2. Instance: An object created from a class; a class defines the structure, and actual programs operate through instances.
    • The main() method serves as an entry point for program execution, and instances must be created using the new() operation to use the class.
    • Variables (members) declared in the class cannot be accessed directly from main(); they must be accessed through instances (instance variables).
    • Accessing variables and methods in instances follows the format of instanceName.variableName and instanceName.methodName.
  3. Variable: Similar to variables in general programming languages, it represents a storage space that can hold one value.
  4. Method: Similar to functions in general programming languages, a function modularizes simple functions, while a method defines the behavior of an object (action).
  5. Comment: Does not affect program execution. It serves as descriptions and notes about the code, enhancing code readability.

Identifier Rules

Identifiers for each component must be written according to rules to avoid conflicts with predefined identifier codes, which already have functions.

Rule Description Example
First Character Must start with a letter, _, or $, and cannot start with a number. validIdentifier, _underscore, $dollar
Other Characters Can consist of letters, numbers, _, or $. identifier1, another_Identifier, $identifier2
Reserved Words Java's reserved words cannot be used as identifiers. int, class, if (not allowed)
Case Sensitivity Java identifiers are case-sensitive. variable, Variable (different identifiers)
No Length Limit There is no limit to the length of an identifier, and spaces are not allowed. thisIsAQuiteLongIdentifier, no SpacesAllowed

These rules must be strictly followed. Additionally, there are conventions to ensure these rules are followed.

Convention Description Example
Class Name Start with a capital noun. Customer, Order, Product
Method Name Start with a lowercase verb. calculateTotal(), getCustomerName(), sendEmail()
Variable Name Start with a lowercase noun. total, customerName, email
Constant Name Uppercase noun. MAX_VALUE, MIN_WIDTH, DEFAULT_TIMEOUT

Types and Declaration of Variables

Variables are like the bricks of a programming language. It's difficult to create meaningful programs without them. Variables represent storage spaces for values and come in various types. They can be broadly divided into two categories.

Primitive Types

Primitive variables are the most basic data types in Java, storing the value itself. There are a total of 8.

Data Type Size Range of Values Literal Representation
byte 8 bits Stores values from -128 to 127. Example: 123
short 16 bits Stores values from -32,768 to 32,767. Example: 12345
int 32 bits Stores values from -2,147,483,648 to 2,147,483,647. Example: 1234567890
long 64 bits Stores values from -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807. Example: 123456789012L
float 32 bits Stores floating point numbers. Range: ±1.4E-45 to ±3.4028235E38. Example: 3.14F
double 64 bits Stores double-precision floating point numbers. Range: ±4.9E-324 to ±1.7976931348623157E308. Example: 3.14159265359
char 16 bits Stores a single 16-bit Unicode character. Range: '\u0000' (0) to '\uffff' (65,535). Example: 'A'
boolean 1 bit Stores values true or false. Example: true, false

Reference Types

Reference types store the memory address of objects. These types include user-defined types such as classes, arrays, and interfaces.

Type Description and Code Example
Class

At the core of object-oriented programming, classes can contain data and methods. Reference type variables point to instances of classes.

public class person {
  string name;
  int age;    
  
  public person(string name, int age) {
      this.name = name;
      this.age = age;
  }
}

public class referencetypeexample {
  public static void main(string[] args) {
      person person = new person("john", 30);
  }
}
Array

An array is a data structure that stores variables of the same type. It has a fixed size and can be accessed via indices.

public class arrayexample {
  public static void main(string[] args) {
      int[] numbers = {1, 2, 3, 4, 5};
  }
}
Interface

Interfaces define a set of methods that a class must implement. Interface type variables can reference objects of classes that implement them.

interface animal {
  void makesound();
}

class dog implements animal {
  public void makesound() {
      system.out.println("woof");
  }
}

public class interfaceexample {
  public static void main(string[] args) {
      animal dog = new dog();
      dog.makesound();
  }
}

Reasons for Distinguishing Between Primitive and Reference Types

Generally, programs read and write variables in memory to store and retrieve values. Java divides the storage space for these variables into stack and heap.

Division Description
Stack

Allocated from high address to low address. Data is stacked from bottom to top, where the last stored data is the first to be retrieved. Thus, it has limited size but allows quick access.
Heap

Allocated in the opposite direction of the stack. It requires memory management (allocating and deallocating variables), has slower access relative to the stack, but allows dynamic allocation.

Utilizing these characteristics, Java stores primitive type variables in the stack, which allows fast access and automatic memory allocation for local variables and method calls. Conversely, it stores reference type variables in the heap, enabling storage of arrays and objects that can be dynamically allocated.

Therefore, when we declare instances or arrays, we are referring not to the value but to the memory location in the heap where the value resides. Consequently, passing values to methods or copying two values may yield different results between reference type and primitive type variables. Variable Lifespan

Variables are stored in memory as explained earlier during compilation. When a program runs, variables are stored in memory, and when the program ends, their memory space is released. During this process, variables are born, exert their influence, and eventually expire like a lifecycle. They are classified based on where they are born and how long they affect the program.

Type Description
Instance Variables
(Instance Variables)
  • Definition: Variables belonging to a class instance
  • Declaration: Inside the class block, outside methods
  • Lifespan: From object creation to garbage collection
  • Access Method: Accessed through objects
public class person {
      // instance variables
      string name;
      int age;
  }
Class Variables
(Class Variables)
  • Definition: Belongs to the class, shared among all instances, using the static keyword
  • Declaration: Inside the class block, outside methods, includes the static keyword
  • Lifespan: From program start to end
  • Access Method: Accessed by class name or objects
public class person {
      // class variable
      static int population;
  }
Local Variables
(Local Variables)
  • Definition: Variables declared inside methods, constructors, or blocks
  • Declaration: Inside methods, constructors, or blocks
  • Lifespan: From block execution to block completion
  • Access Method: Accessed only within the declared block
public void display() {
      // local variable
      int count = 0;
      system.out.println(count);
  }
Parameters
(Parameters)
  • Definition: Variables declared in the parameter list of methods or constructors
  • Declaration: In the method or constructor declaration
  • Lifespan: From method call to completion
  • Access Method: Accessed within the method or constructor
public void setname(string name) {
      // parameter
      this.name = name;
  }

Operators

Operators are symbols used in programming to perform operations and produce results during the process of computations.

  1. Arithmetic Operators
    Operator Description Example
    + Addition int sum = 5 + 3;
    - Subtraction int difference = 5 - 3;
    * Multiplication int product = 5 * 3;
    / Division int quotient = 6 / 3;

  2. Comparison Operators
    Operator Description Example

    > Greater than boolean isGreater = (x > y);
    < Less than boolean isLess = (x < y);

    <= Less than or equal to boolean isLessOrEqual = (x <= y);
  3. Logical Operators
    Operator Description Example

    || Logical OR boolean result = (condition1 || condition2);
    ! Logical NOT boolean result = !condition1;
  4. Assignment Operators
    Operator Description Example
    = Assignment int x = 5;
    += Addition and assignment x += 3;
    -= Subtraction and assignment x -= 3;
    *= Multiplication and assignment x *= 3;
    /= Division and assignment x /= 3;
    %= Modulus and assignment x %= 3;
  5. Bitwise Operators
    Operator Description Example
    & Bitwise AND int result = x & y;
    | Bitwise OR int result = x | y;
    ^ Bitwise XOR int result = x ^ y;
    ~ Bitwise NOT int result = ~x;
    << Left shift int result = x << 1;
    >> Right shift int result = x >> 1;
  6. Unary Operators
    Operator Description Example
    + Unary plus int result = +x;
    - Unary minus int result = -x;
    ++ Increment x++;
    -- Decrement x--;
    ! Logical NOT boolean result = !condition;
    ~ Bitwise NOT int result = ~x;
